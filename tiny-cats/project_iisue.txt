### Step 1: TypeScript install karo

Project ke root folder me:

```bash
npm install --save-dev typescript
```

Ya globally:

```bash
npm install -g typescript
```

Lekin project-level installation better hai.

```bash
npx tsc --version
```

Output kuch aisa aana chahiye:


### Step 3: tsconfig generate karo

Ab ye command chalegi:

```bash
npx tsc --init
```

Isse `tsconfig.json` create ho jayegi.

An import path can only end with a '.ts' extension when 'allowImportingTsExtensions' is enabled.



Ye TypeScript ka common error hai.

### Error

```ts
import app from "./app.ts";
```

Agar tum `.ts` extension ke saath import karoge to TypeScript by default error dega:

```text
An import path can only end with a '.ts' extension when
'allowImportingTsExtensions' is enabled.
```

---

### Solution 1 (Recommended)

`.ts` extension hata do:

```ts
// ❌
import app from "./app.ts";

// ✅
import app from "./app";
```

Yehi standard TypeScript/Node.js practice hai.

---

### Solution 2

Agar tum explicitly `.ts` use karna chahte ho to `tsconfig.json` me:

```json
{
  "compilerOptions": {
    "allowImportingTsExtensions": true
  }
}
```

Lekin ye tabhi use hota hai jab tum specific ESM setup use kar rahe ho. Normally iski zarurat nahi padti.

### Tiny Cats Backend ke liye

Agar structure kuch aisa hai:

```text
src/
 ├── server.ts
 ├── app.ts
 ├── routes/
 │    └── cat.routes.ts
```

To imports aise hone chahiye:

```ts
import app from "./app";
import catRoutes from "./routes/cat.routes";
```

**`.ts` mat lagao.**

---

Agar apna `tsconfig.json` aur jis line par error aa raha hai wo import statement bhej do, to main exact bata dunga ki CommonJS use kar rahe ho, ESM use kar rahe ho, ya NodeNext configuration ki wajah se error aa raha hai.



Relative import paths need explicit file extensions in ECMAScript imports when '--moduleResolution' is 'node16' or 'nodenext'. Did you mean './src/app.js'?


Ab jo error aa raha hai usse lag raha hai ki tumhare `tsconfig.json` me:

```json
{
  "module": "NodeNext",
  "moduleResolution": "NodeNext"
}
```

ya

```json
{
  "module": "Node16",
  "moduleResolution": "Node16"
}
```

set hai.

NodeNext/Node16 me TypeScript compile hone ke baad Node actual `.js` files run karta hai, isliye imports me `.js` extension likhna padta hai.

### Agar tumhara code hai:

```ts
import app from './src/app';
```

To ise karo:

```ts
import app from './src/app.js';
```

Dhyan do:

* Source file `app.ts` hi rahegi.
* Import me `.js` likhoge.
* TypeScript compile karke `app.js` generate karega.

---

### Example

**app.ts**

```ts
import express from 'express';

const app = express();

export default app;
```

**server.ts**

```ts
import app from './app.js';

app.listen(3000, () => {
  console.log('Server running');
});
```

### Alternative (simpler for beginners)

```json
  }
}
```

kar do.

Phir imports normal rahenge:

```ts
import app from './src/app';
```

aur `.js` extension ki zarurat nahi padegi.

### Mujhe lag raha hai tum NodeNext aur ts-node ko mix kar rahe ho.

#### Option 1 (Recommended for learning TypeScript + Express)

`tsconfig.json` me ye use karo:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "CommonJS",
    "moduleResolution": "node",
    "esModuleInterop": true,
    "strict": true,
    "skipLibCheck": true
  }
}
```

Aur imports:

```ts
import app from './src/app';
```

`package.json`

```json
{
  "scripts": {
    "dev": "ts-node server.ts"
  }
}
```

Ye sabse simple setup hai.

---

#### Option 2 (NodeNext / ESM)

Agar `package.json` me:

```json
{
  "type": "module"
}
```

hai aur NodeNext use karna hai, to normal `ts-node` nahi chalega.

Script honi chahiye:

```json
{
  "scripts": {
    "dev": "node --loader ts-node/esm server.ts"
  }
}
```

Aur imports:

```ts
import app from './src/app.js';
```

**package.json**

**server.ts**

Mujhe 99% lag raha hai ki tumhare `package.json` me `"type": "module"` hai aur tsconfig NodeNext par hai. Main exact configuration bata dunga aur 2 minute me fix ho jayega.
